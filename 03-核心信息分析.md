# 03 · 核心信息分析

> 按四条主线归纳。编号 [#n] 对应 [02-信息来源记录.md](02-信息来源记录.md) 中的信息源。

## 主线一：编程 Agent 的能力演进——"补全工具"已死，"自主 Agent"当立

**事实层：**

- 主流产品（Claude Code、Cursor、Copilot、Codex、Antigravity）已全部具备：自主读代码库、跨文件修改、运行测试、调试、提交 PR 的完整闭环能力 [#10][#12]。
- 官方 SWE-bench Verified 榜单（统一 harness 重测，2026-02）：头部模型集中在 70-77% 区间，**第 1 名与第 8 名仅差约 6 分** [#6][#7]。
- 开源侧同样成熟：OpenHands ~75K star、Cline ~62K star、OpenCode ~165K star，能力覆盖沙箱自主运行、CI/CD 无头模式、MCP 工具调用 [#8][#9][#10]。

**分析层（我的判断）：**

1. **模型能力差距正在收敛，"选哪个模型"越来越不重要，"怎么用"越来越重要。** 6 分的榜单差距在真实工程中几乎感知不到，但 harness（工作流、上下文管理、工具配置）的差距是数量级的——arXiv 论文 [#5] 把这称为 "Harness Engineering"，我认为这是这半年最被低估的概念。
2. **竞争焦点从"能力"转向"工作流形态"**：IDE 内嵌（Cursor）vs 终端原生（Claude Code）vs 云端自主（Codex/Jules）。这说明赛道从技术竞争进入产品竞争阶段，对个人来说意味着：学"某个工具的用法"贬值快，学"Agent 工作流的通用方法论"保值。

## 主线二：计费剧变——补贴时代结束，Token 成为新的成本单位

**事实层：**

- GitHub Copilot 2026-06-01 起全面转向 AI Credits 按量计费（1 Credit ≈ $0.01），代码补全保持免费，Agent 模式按 Token 扣费 [#1]。
- 极端案例：有用户月费从 $50 预估涨至 $3000（60 倍）[#13]（注意：这是用户晒单的二手信息，代表尾部而非中位数）。
- DX 对 400+ 企业的研究：AI 编程工具带来的 PR 吞吐量**中位增益仅 7.76%**，远低于厂商宣传的 3 倍；实际人均成本 $200-600/月 [#11]。
- 国内同步跟进积分制：腾讯云代码助手标准版 99 元/月 2000 积分 [#3]；Qoder CN 个人 Pro 59 元/月 [#4]。

**分析层（我的判断）：**

1. **"账单暴涨 60 倍"是尾部噪音，"按量计费成为行业共识"才是信号。** 极端案例会上头条，但真正值得记住的是结构性变化：AI 编程成本从固定支出变成浮动支出，会用的人和不会用的人成本差 10 倍以上。
2. **7.76% 这个数字比任何厂商发布会都重要。** 它说明工具本身不自动带来生产力，中位数团队用得很差。这反过来证明："会高效使用 Agent"是稀缺技能，值得投入学习。
3. **成本敏感场景存在明确套利空间**：DeepSeek API 约 2 元/百万 Token [#14]，配合开源 Agent（OpenHands/Cline）可以用几十元/月复现大厂 $200/月方案的大部分能力。这是学生党最该知道的信息差。

## 主线三：开源与国产方案——个人使用门槛正在坍塌

**事实层：**

- 中国 AI 编程市场：2025 年约 3.99 亿元 → 2026 预计 11.73 亿元，阿里以 47.6% 份额居首（IDC 数据，via 极客公园）[备查]。
- 通义灵码升级为 Qoder CN 产品矩阵，其 CLI 以 Apache 2.0 开源，对标 Claude Code [#4]。
- 国产模型真实工程横评（雷峰网，2026-06）：MiniMax M3 综合 85.3 分，Kimi K2.6 82.5，Qwen 3.7 Max 82.2，DeepSeek V4 Pro 78.6 [#14]。
- 开源 Agent 生态分层清晰：OpenHands（沙箱自主/CI）、Aider（Git 原生编辑，但 2025-08 后无新 release，维护放缓）、Cline（VS Code + MCP 原生）[#8][#9][#10]。

**分析层（我的判断）：**

1. **"国产 = 落后"的旧地图已经失效。** 国产模型在 SWE-bench 上进入 70+ 区间（DeepSeek V3.2 70.0%、Kimi K2.5 70.8% [#6]），与国际一线的差距缩小到"工程可接受"范围，而价格是零头。
2. **注意开源项目的"维护健康度"信号**：Aider 曾是明星项目（~45K star），但近一年无 tagged release，社区动能转向 OpenCode 等新项目 [#10]。Star 数是滞后指标，release 频率才是领先指标——这是筛选开源信息的重要方法。

## 主线四：标准化——MCP 与 AGENTS.md 定义了新的"基础设施层"

**事实层：**

- 2025-12，Anthropic 将 MCP 捐入 Linux 基金会旗下 AAIF（OpenAI、Google、Block 联合创始），4 个月内 170+ 成员组织加入 [#15]。
- MCP SDK 月下载量约 1 亿次（2024-11 发布时仅 200 万）；公开 MCP Server 超万个；MCP 1.0 于 2026-06-24 正式发布 [#15]。
- AGENTS.md 经 2853 个仓库的实证研究确认为跨工具（Claude Code/Cursor/Copilot）配置 Agent 行为的事实标准 [#5]。

**分析层（我的判断）：**

1. **协议中立化是生态成熟的标志性事件。** 类比 Kubernetes 捐入 CNCF：当协议不再属于单一厂商，围绕它构建的技能和项目才有长期价值。现在学 MCP 开发，风险已经显著低于一年前。
2. **标准化创造了新的"卖铲子"机会**：写 MCP Server、做 Agent 配置工程（AGENTS.md/规则文件）、Agent 安全审计——这些在标准确立后才成立的细分方向，目前竞争者少、需求增长快，是典型的早期机会窗口。
