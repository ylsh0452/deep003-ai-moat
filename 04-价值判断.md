# 04 · 价值判断：真信号 vs 噪音

> 信息雷达的核心不是收集，而是过滤。以下是我用来区分信号与噪音的五条标准，以及本次收集信息的分类结果。

## 我的五条过滤标准

1. **一手优先**：官方公告/文档/论文/可复现基准 > 媒体转述。媒体标题党（"暴涨 60 倍"）背后往往有一个更平淡但更重要的一手事实（"计费模式转型"）。
2. **结构性 > 事件性**：会改变"游戏规则"的信息（计费模式、协议标准）比"某产品发布新版本"更值得记录。
3. **有数字、有方法的 > 有观点的**：DX 的 7.76%（样本 400+ 企业、方法公开）比一百篇"AI 让效率提升 10 倍"的软文更有价值。
4. **反直觉的 > 顺直觉的**：与主流叙事相反且有据可查的信息（如"中位增益仅 7.76%"）往往是最大的信息差所在。
5. **可行动的 > 仅供谈资的**：一条信息如果推不出任何行动（学什么/用什么/做什么/避开什么），它的价值上限就是社交谈资。

## 分类结果

### ✅ 强信号（结构性变化，值得据此调整行动）

| 信息 | 为什么是强信号 |
|---|---|
| Copilot 等全面转向按量计费 [#1][#12] | 官方一手 + 全行业跟进，改变成本结构，直接影响个人和团队的工具选型逻辑 |
| MCP 捐入 Linux 基金会、1.0 发布 [#15] | 协议中立化 = 生态锁定风险大幅下降，围绕 MCP 的技能投资从"赌注"变成"基建" |
| SWE-bench 头部模型分差收敛到 ~6 分 [#6] | 统一 harness 重测的结果，说明"模型选择焦虑"可以放下，重心应移向工作流 |
| PR 吞吐量中位增益仅 7.76% [#11] | 大样本 + 反直觉，戳破厂商叙事，同时证明"会用 Agent"是稀缺技能 |
| AGENTS.md 成为跨工具事实标准 [#5] | 2853 仓库实证，学一次到处能用，学习投资回报率高 |

### ⚠️ 中等信号（值得跟踪，但需持续验证）

| 信息 | 保留意见 |
|---|---|
| 国产模型横评排名（MiniMax 85.3 分等）[#14] | 单一媒体的评测口径，任务样本小，排名参考价值 < 「国产已进入可用区间」这个整体结论 |
| 中国市场 4 亿 → 11.7 亿的预测 [备查] | IDC 预测有方法论，但"预计"就是预计，方向可信、具体数字不必当真 |
| Aider 维护放缓、OpenCode 崛起 [#10] | 开源项目动能变化快，需要每月看一次 release 频率再下结论 |

### ❌ 噪音（记录下来是为了展示如何识别）

| 信息 | 为什么是噪音 |
|---|---|
| "有人月费冲到 $3000" [#13] | 尾部个案 + 用户自述截图，无法验证其使用方式是否合理；情绪价值 > 信息价值 |
| "AI 淘汰初级程序员"类标题 [备查] | 就业结构变化是真的，但"淘汰"叙事既不精确也推不出个人行动；有效表述是"纯执行型编码技能贬值、架构/审查/AI 失败模式识别能力溢价" |
| 各厂商发布会里的"3 倍效率提升" | 与 7.76% 中位数矛盾，属于厂商最优场景宣传，采信一手研究而非发布会 |

## 一个交叉验证的实例

**声明**："Claude Code 在 SWE-bench Verified 达到 88.6%"（出自厂商相关介绍，via [#12] 类综述）。
**交叉验证**：官方统一 harness 榜单 [#6] 中最高分是 76.8%（Claude 4.5 Opus）。
**解释**：两个数字都"真"，但口径不同——厂商用自家优化的 agent harness + 更宽松设置，官方榜用统一最小 harness。**结论**：跨来源比较基准分数时必须先对齐评测口径，否则比较无意义。这是本次任务中我踩到又爬出来的最有价值的一个坑。
