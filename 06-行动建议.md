# 06 · 行动建议

> 每条建议都必须能回答"具体做什么、为什么现在做、依据是哪条信息"。按四个场景展开。

## 场景一：学习

| 行动 | 具体做法 | 依据 |
|---|---|---|
| 把学习重心从"工具用法"转向"Harness 工程" | 给自己的练习项目写 AGENTS.md / 规则文件，练习上下文管理和任务拆解；同一任务在两个不同 Agent 里跑，对比差异 | AGENTS.md 是跨工具标准 [#5]；模型分差收敛后 harness 是主要变量 [#6] |
| 建立"生成代码必审查"的习惯 | 每次让 Agent 写完代码，强制自己解释每个改动点后再提交 | 中位增益仅 7.76% 说明多数人用得差 [#11]；审查能力是溢价技能 |
| 低成本配置一套个人 Agent 环境 | 开源 Agent（Cline 或 OpenHands）+ 低价 API（DeepSeek 约 2 元/百万 Token），预算控制在 50 元/月内 | 开源方案成熟 [#8][#9]；国产 API 性价比 [#14] |

## 场景二：比赛

| 行动 | 具体做法 | 依据 |
|---|---|---|
| 报名 1-2 个 Agent 方向比赛 | 优先级：火山杯 Agent 创新大赛（高校赛道、工具链门槛低，Q3 进行中）→ GOAI 开源大赛（9-22 杭州决赛，Agent 基础设施赛道） | 官方赛程 [#2]、GOAI 公告 [备查] |
| 用比赛倒逼作品化 | 参赛作品直接以开源仓库形式开发（GOAI 要求默认开源），赛后成为简历项目 | GOAI"可运行、可验证、可共建"的评审导向 |
| 赛前准备清单 | 熟练一套 Agent 开发栈（如 MCP Server 开发 + 一个开源 Agent 框架）；准备可复用的项目脚手架 | MCP 已是中立标准，比赛和就业通用 [#15] |

## 场景三：项目

| 行动 | 具体做法 | 依据 |
|---|---|---|
| 做一个解决自己真实痛点的 MCP Server | 例如：把学校教务系统/课程表/文献库包装成 MCP Server，接入日常使用的 Agent | 公开 MCP Server 破万但长尾场景远未覆盖 [#15]；早期生态窗口 |
| 给现有开源项目贡献 Agent 配置 | 给自己常用的开源项目提交 AGENTS.md 或改进其 Agent 兼容性，成本低、可见度高 | 2853 仓库已采用，但多数仓库还没有 [#5] |
| 项目中引入成本监控 | 任何用到 LLM API 的项目，从第一天就记录 Token 消耗，做成本仪表盘 | 按量计费时代成本治理是刚需 [#1][#12] |

## 场景四：个人成长与未来创业

| 行动 | 具体做法 | 依据 |
|---|---|---|
| 把本仓库变成长期信息雷达 | 固定节奏更新（见 07 复盘中的运转机制），每月一次增量 commit，形成公开可查的积累记录 | 信息雷达的价值在持续性，单次报告只是起点 |
| 关注"卖铲子"型创业方向 | 值得跟踪的三个细分：Agent 成本治理工具、垂直场景 MCP Server、Agent 安全/审计。每月检查这些方向的新项目和融资 | 计费剧变创造成本治理需求 [#1][#11]；标准化创造生态位 [#15] |
| 警惕的方向 | 通用 Agent 框架（巨头和明星开源项目已占满）、纯套壳工具（无壁垒） | 六大平台能力趋同 [#10][#12]；OpenCode 等已占开源生态位 |

## 优先级排序（如果只做三件事）

1. **本月内**：用开源 Agent + 低价 API 搭好个人环境，开始每天实际使用（学习成本最低、复利最高）
2. **两个月内**：完成一个 MCP Server 小项目并开源（作品 > 证书）
3. **三个月内**：带着作品参加火山杯或 GOAI（用比赛验证并放大前两步的成果）
