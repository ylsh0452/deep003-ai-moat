# AI Token 成本仪表盘

> Claude Code 真实问题解决任务 —— 将手动 Token 消耗追踪转化为可视化仪表盘。

## 是什么

一个纯前端网页仪表盘，用于记录和可视化 AI API 的 Token 消耗与成本。打开即可用，数据存储在浏览器 localStorage 中。

## 功能

- **统计卡片**：总调用次数 / 总 Token / 总成本 / 日均成本
- **趋势图**：每日 Token 消耗折线图
- **成本分布**：按模型的饼图
- **使用记录表**：完整的调用历史
- **手动录入**：表单添加新的 API 调用记录
- **内置示例数据**：含 5 条真实记录（2026-07-18 实验数据）

## 使用方式

```

```

### 方式二：命令行记录 Token

```bash
node log.js "修复了搜索排序 bug" 320 1450 deepseek-v4-pro
```

## 为什么要做这个

按量计费时代，Token 消耗直接影响成本。做一个仪表盘不是目的——**养成记录和审视 Token 消耗的习惯**才是。这个习惯的直接后果是：

- 知道每次 Agent 调用花了多少钱
- 能对比不同模型的实际性价比
- 月预算 ¥50 是否真的够用 → 有数据支撑而不是靠感觉
