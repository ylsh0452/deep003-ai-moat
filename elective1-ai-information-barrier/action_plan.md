## 1. 这次信息整理后，我最应该关注哪个方向？

**Harness Engineering——也就是"怎么用好 Agent"，而不是"哪个 Agent 更好"。**

理由：SWE-bench 头部模型只差约 6 分 [#6]，模型选择不再是瓶颈。但同一模型在不同用户手里的产出差距是天壤之别——中位团队 PR 增益仅 7.76%，而顶尖团队可能高出 10 倍以上 [#11]。这中间的差距就是 harness 能力（AGENTS.md 配置、上下文管理、任务拆解、代码审查）。

具体来说，我会把关注重点从"又有什么新模型发布了"转移到三件事上：
- 怎么给我的项目写 AGENTS.md 才能让 Agent 真正理解我的意图
- 怎么把一个复杂任务拆成 Agent 能稳定执行的小步
- 怎么审查 Agent 生成的代码，而不是盲目信任

## 2. 我现在最应该学习或尝试哪个工具？

**Cline（VS Code 插件）+ DeepSeek API（≈¥2/百万 Token），总成本控制在 ¥50/月以内。**

理由：
- Cline 是 VS Code 原生插件，安装即用，不需要学命令行；MCP 原生支持，后续做 MCP Server 项目可以无缝衔接 [#9]
- DeepSeek API 的价格是国际方案的几十分之一，国内直连稳定，不需要代理 [#14]
- 这个组合完全合规（DeepSeek 官方 Anthropic 兼容端点），不存在灰色渠道风险
- 我已经在本次任务的实验中验证了这个方案的可行性（experiment/ 目录下的 bug 修复实测：631 tokens、< ¥0.01）

**不选** Claude Code CLI 的原因：CLI 门槛更高，目前在网络和付费上也有额外障碍。等我用熟 Cline 之后，如果确实需要终端场景再做迁移，那时候 harness 能力已经有了，迁移成本很低。

| 方向 | 为什么有机会 | 我现在能做什么 |
|---|---|---|
| **Agent 成本治理工具** | 按量计费后，企业面临"Token 预算不可控"的痛点，而市场上专门的 Agent 成本治理工具极少 [#1][#11][#12] | 做最小原型：一个统计 Agent 调用次数和 Token 消耗的简单仪表盘 |

## 为什么是这个顺序

```
7 月：打基础（搭环境 + 写第一个 AGENTS.md）
  ↓
8 月：出作品（MCP Server 开源 + 给项目贡献 AGENTS.md）
  ↓
9 月：验证（参赛 → 用比赛反馈迭代作品 → 成为简历项目）
```

这个顺序的逻辑是：**环境 → 习惯 → 作品 → 外部验证**。每一步都是下一步的燃料，且每一步都不依赖灰色渠道或大额预算。
