# 07 · 任务复盘

## 1. 这次任务让我发现了哪些信息差？

做完这次信息整理后，我发现了三个之前完全不知道的信息差：

**第一个：AI 编程工具的实际效率提升远低于我的想象。** 我之前默认以为用了 AI 编程工具效率就是翻倍的，但 DX 对 400+ 企业的研究显示中位 PR 吞吐量增益只有 7.76%。这说明大部分人（包括以前的我）在用 AI 编程工具时并没有真正发挥它们的作用——只是让它们写代码，但没有掌握上下文管理、任务拆解和代码审查。

**第二个：国产模型和开源 Agent 已经足够好用，价格是国际方案的几十分之一。** 我之前对国产 AI 编程工具的认知停留在"不如 Claude/GPT"的模糊印象里，不知道 DeepSeek 已经在 SWE-bench 上达到 70% 以上，也不知道 Qoder CN CLI 已经以 Apache 2.0 开源。这意味着我一直花着不必要的钱或忍受着不必要的不便——其实几十元/月就能搭一套生产级方案。

**第三个：MCP 已经从 Anthropic 的项目变成了行业标准，现在学是安全性最高的时机。** 我之前不知道这个协议的存在，更不知道它已经成为跨所有编程 Agent 的通用标准。知道之后立刻意识到：这不是"学一个工具"，这是"学一个基础设施"，两者的投资回报完全不在一个量级。

## 2. 我之前是否存在只看二手总结、短视频或社群消息的问题？

存在，而且程度比我愿意承认的更严重。

在这次任务之前，我对 AI 动态的了解方式基本是：(1) 刷到微博/B站上的"AI 又又又更新了"类视频 (2) 在群里看到别人转发的"今天 AI 圈发生了什么"总结 (3) 偶尔点开一两篇公众号文章但只看标题和加粗部分。

这种方式的问题：我接收的已经是经过两层甚至三层转述的信息——厂商发布会 → 科技媒体 → 自媒体总结 → 社群压缩成一句话。每一层都在丢失细节、放大情绪。如果不是这次任务让我回到原始链接逐条核实，我不会发现 SWE-bench 的"88.6%"和"76.8%"竟然都是对的——只是口径不同。而这条信息在二手总结里根本不会出现，它们只会写"Claude Code 在编程基准上排名第一"然后配上几个火焰 emoji。

这次任务的检索方法——搜原始链接、核实一手数据、交叉对比、标注来源类型——对我是一个有意识的矫正。我不可能以后每条信息都做到这个程度（时间不允许），但我至少知道了"回去看一手来源"这个动作的存在，也知道了二手总结会丢失什么。

## 3. 我是否学会了区分真实机会和短期热点？

学会了基本方法，但需要持续练习。

我建立了一套五条过滤标准：一手优先 / 结构性 > 事件性 / 有数字有方法的 > 有观点的 / 反直觉的 > 顺直觉的 / 可行动的 > 仅供谈资的。这次任务中严格按标准走了一遍，效果是明显的——比如我识别出"有人月费暴涨到 $3000"是噪音、"按量计费成为行业共识"才是信号。

但目前这套标准还只运行了一次，没有经过"时间验证"这一关。三个月后回头看，我当时的"强信号"判断是否被验证是正确的？哪些"中等信号"升级了？哪些我认为是噪音的东西其实有道理？这些都需要季度复盘来校准。

还有一个诚实的问题：我能做到是因为这次任务是"交作业"——有评分压力。没有评分压力的时候，我还会不会花时间做这件"看不到即时收益"的事？这是我需要在持续运转中用行动回答的问题。

## 4. 我最有价值的一条信息是什么？为什么？

**最有价值的一条是 DX 研究报告中的"PR 吞吐量中位增益仅 7.76%"（来源 #11）。**

理由不是这个数字本身，而是这个数字展示出的信息结构：

同一件事——"AI 编程工具能提升效率吗"——厂商说"3 倍"，独立研究说"7.76%"，差距是 38 倍。这个差距本身才是真正的信息。它说明：(1) 厂商有动机夸大，(2) 大多数用户确实用得差，(3) "会用"和"不会用"之间有巨大的套利空间。

这条信息还直接改变了我后续的行动方向——如果中位增益真的只有 7.76%，那我应该花时间去研究那顶尖的一小撮人做了什么不一样的事（答案是 harness engineering），而不是把更多钱砸在更贵的订阅档位里。这就是"一条信息 → 改变一个决策"的实例。

## 5. 哪条信息看起来很热，但我认为暂时不适合自己？

**本地部署开源模型（Ollama / llama.cpp 等）。**

看起来热是因为：社区里很多人讨论"把模型跑到本地""数据不出你的电脑""彻底摆脱 API 付费"。这个方向本身有价值——对隐私敏感的企业、对需要离线场景、对有硬件条件的人来说是合理选择。

但对我暂时不适合，原因是：硬件门槛（需要 GPU，我没有）+ 学习成本高（模型格式、量化、推理引擎，每一层都是新知识）+ API 价格已经足够低（¥2/百万 Token 对学生来说是零钱级别）。在 API 价格的三十分之一和本地部署的三十倍麻烦之间，我选前者。不是永远不做，是**现在不做**——等到我有了 GPU 硬件或者 API 价格大幅上涨的时候，再做也不迟。

这个判断练习让我意识到：信息雷达不只是"发现好东西"，也是"识别不适合自己的好东西"。后者的重要性被严重低估了。

## 6. 我接下来会如何建立长期 AI 信息雷达？

基于这次任务的实践，我设定了一个**周-月-季三层节奏**：

**每周（30 分钟）**：快速扫描固定信源清单——官方博客（GitHub Blog、Anthropic Blog）、开源仓库 Release 页（OpenHands、Cline）、高信噪比博客（Simon Willison）、定价页面。只记"结构性变化"（如价格调整、协议变更、重大版本发布），忽略"某产品又更新了 0.0.x"类的信息。

**每月（1-2 小时）**：深读当月最重要的 1 个变化，做一次口径核查 + 交叉验证。同时把一条"可行动"的信息变成最小实验——装一个工具、跑一个 API、写一个 Demo，把结果记录回这个 GitHub 仓库。实验不需要正式，但必须真实。

**每季（2-3 小时）**：回头看本季度我在"强信号"中标记的那些信息——它们真的发生了吗？我据此做出的行动是正确的吗？据此校准过滤标准。

判断是否值得记录的快速心法：**这条信息一年后还成立吗？如果成立，它改变我的哪个决定？** 两问都答不上来的，直接丢弃。

这个机制的关键不是"做得多"，而是"做了什么都有痕迹"——所有更新都会 commit 到这个 GitHub 仓库，形成公开可查的信息雷达运行记录。

## 7. 这个任务如何帮助我和普通 AI 使用者拉开差距？

普通 AI 使用者：刷到 AI 新闻 → 觉得"好厉害"→ 下一篇 → 偶尔焦虑"我是不是又落后了"→ 继续刷下一篇。

完成这个任务后，我不再是那个循环里的人了。具体的差距体现在三个层面：

**信息质量层**：我知道怎么区分一手和二手信息，知道去看原始链接、查评测口径、找独立研究——而不只是看别人总结好的"今天 AI 圈十大新闻"。至少 80% 的 AI 使用者永远不会做这件事。

**判断力层**：我有一套明确的过滤标准，能说出"为什么这是信号、为什么那是噪音"。当同学转发一条"AI 又淘汰了一个职业"的文章时，我可以快速判断这是一条情绪内容还是一份有方法论的研究——而不是被焦虑带着走。

**行动力层**：信息 → 判断 → 行动，每一步都有依据。我知道我下个月要做什么（装 Cline + 搭环境）、再下个月要做什么（MCP Server + 开源），也知道这些行动和我的比赛、项目和长期方向之间的关系。而普通使用者的"行动"通常止于"把文章收藏起来以后再看"。

一句话：普通 AI 使用者在消费信息，我在建信息雷达。前者永远被动，后者开始主动。这个任务的意义不在于交了一份作业，而在于我第一次完整地跑了一遍"固定信源 → 明确过滤 → 强制行动"的闭环。闭环一旦建立，后面的每一次更新都在拉开差距。本仓库就是这个闭环的第一个里程碑。
