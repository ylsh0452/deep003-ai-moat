# 05 · 风险边界

> 信息雷达如果不标注失效条件，就会从资产变成负债。以下是本报告结论的适用边界和已知风险。

## 1. 信息时效性风险

- **半衰期极短**：本仓库快照日期为 2026-07-18。定价（如腾讯云 99 元/月、Copilot Credits 费率）历史上以季度为单位变化，**任何超过 3 个月的价格信息都应视为过期**。
- **榜单漂移**：SWE-bench 分数随模型迭代每月都在变，本报告记录的是"分差收敛"这一趋势，而非具体名次。
- 应对：02 文件中每条信息都标注了时间；README 中声明快照属性。

## 2. 来源偏差风险

- **厂商宣传偏差**：厂商基准分数普遍使用自家最优 harness（见 04 中 88.6% vs 76.8% 的口径差异案例）。凡厂商自报数据，一律标注并寻找第三方复测。
- **媒体幸存者偏差**："$3000 账单"上头条，"绝大多数人账单没变化"不上头条。二手媒体信息只用于发现线索，不用于支撑结论。
- **英文信息时滞**：国产工具的英文报道普遍滞后 1-2 个月，中文一手信源（官方文档、开发者社区）不可省略。

## 3. 依赖与锁定风险

- **工具锁定**：深度绑定单一厂商工作流（如只会用某家 IDE 的私有配置）的技能贬值风险高。缓解：优先学跨工具标准（MCP、AGENTS.md），它们在厂商之间可迁移。
- **成本失控**：按量计费下 Agent 长时间自主运行可能产生不可预测账单（尾部案例见 [#13]）。缓解：个人使用设置硬性额度上限；优先用有封顶的订阅制或低价 API（DeepSeek 约 2 元/百万 Token）。
- **网络与合规**：部分国际工具在国内直连不稳定（检索中多个国内评测提到 Claude API 直连成功率问题），选型时必须把可用性纳入考量，而不是只看榜单。

## 4. 学习误区风险

- **误区一：追新工具 = 学习**。工具半年一换代，追逐每个新产品是伪学习。真学习对象是不变量：prompt 工程、上下文管理、任务拆解、代码审查能力。
- **误区二：AI 写得出 = 我会了**。DX 数据（中位增益 7.76%）暗示大多数人高估了自己从工具中获得的收益。不做代码审查、不理解生成代码的使用方式，会制造"能力幻觉"。
- **误区三：信息收集 = 信息优势**。收藏夹里的 100 篇文章不构成信息雷达；有过滤标准、有行动转化的 10 条信息才构成。

## 5. 本报告自身的边界

- 检索范围以中英文公开网络信息为主，未覆盖付费报告、闭门社区（如部分 Discord/内部群）的信息。
- 分析中所有"我的判断"均为基于当前信息的推断，置信度不一，已在行文中与事实陈述区分。
- 竞赛信息（GOAI、火山杯等）的报名条件与截止时间以官方页面实时信息为准，本报告不承担时效责任。
