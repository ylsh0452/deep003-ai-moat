# 01 · 主题选择：AI 编程工具与 Agent

## 为什么选这个主题

从三个标准筛选主题：**与我相关、变化够快、信息差够大**。

1. **与我相关**：编程工具是学习者每天都在用的东西，信息差直接转化为效率差和成本差。同样是 20 美元/月的预算，知道怎么选工具的人和不知道的人，产出差距可能是数倍。
2. **变化够快**：2026 年上半年这个赛道发生了三件结构性事件——GitHub Copilot 全面转向按量计费（6 月 1 日）、MCP 1.0 正式发布（6 月 24 日）、国产工具市场规模预计从 4 亿冲向 11.7 亿元。半年内格局重写，信息雷达的价值最大化。
3. **信息差够大**：大部分人对 AI 编程工具的认知还停留在"Copilot 补全代码"，而实际上前沿已经进入"Agent 自主完成整个 issue → 提交 PR"的阶段。这个认知差就是机会。

## 聚焦范围（明确不做什么）

- ✅ 做：编程 Agent 产品（Claude Code、Cursor、Copilot、Codex 等）、开源编码 Agent（OpenHands、Aider、Cline）、国产方案（Qoder、DeepSeek、Trae）、基准测试（SWE-bench）、计费与成本、协议标准（MCP/AGENTS.md）、相关比赛机会
- ❌ 不做：通用大模型军备竞赛细节、多模态生成、AI 绘画/视频——不是不重要，而是聚焦才有深度

## 我的信息雷达定义

信息雷达 = **固定的信源清单 + 固定的扫描频率 + 固定的过滤标准**。

| 层级 | 信源类型 | 扫描频率 | 举例 |
|---|---|---|---|
| 一手 | 官方博客/发布公告/定价页 | 每周 | github.blog、anthropic.com、阿里云开发者社区 |
| 一手 | 开源仓库 Release/Star 趋势 | 每周 | OpenHands、Cline、Aider 的 GitHub |
| 一手半 | 基准测试榜单 | 每月 | SWE-bench Verified、Terminal-Bench |
| 二手 | 高信噪比个人博客/独立分析 | 每周 | Simon Willison 博客、DX 研究报告 |
| 二手 | 行业媒体/社区 | 有事件时 | InfoQ、极客公园、雷峰网、掘金 |
| 机会 | 比赛/活动聚合页 | 每月 | 火山引擎大赛页、高校竞赛通知 |

过滤标准见 [opportunity_judgement.md](opportunity_judgement.md)。

## 这个主题和我的 AI 壁垒方向有什么关系

我的 AI 壁垒方向是**信息获取与工程落地的结合**——不仅要能发现信息差，还要能把它转化为实际产出（项目、作品、比赛成果）。AI 编程工具是这个壁垒方向的核心基础设施：它既是我追踪的"信息对象"，也是我做所有其他事情的"效率杠杆"。掌握这些工具的最新动态，意味着我能用更低的成本、更快的速度把想法变成可运行的代码和可展示的作品，而大多数人还在被动等别人总结或花高价用老方案。

## 对学习、比赛、项目、创业的具体帮助

- **学习**：知道哪个工具适合什么场景，避免花时间学即将过时的工具；用 Agent 辅助理解复杂代码库，学习效率远超手动阅读。
- **比赛**：GOAI 开源大赛、火山杯 Agent 大赛等比赛本身就围绕 Agent 技术展开，提前建立信息雷达可以直接转化为参赛选题和工具链优势。
- **项目**：GitHub 项目从空仓库到有可展示成果的周期，使用 Agent 后可以缩短到原来的几分之一；MCP Server 开发、AGENTS.md 配置等细分方向还有大量未覆盖的需求。
- **创业**：Agent 成本治理、垂直场景 MCP Server、Agent 安全审计——这三个方向在 2026 年仍处于早期窗口，竞争者少，是我长期跟踪的潜在切入点。

## 我目前对这个主题的了解程度

在本次信息雷达任务之前，我的了解停留在"用过一些工具、看过一些推荐文章"的层面——知道 Copilot 补全很强、Claude 写代码不错，但对开源 Agent 生态、SWE-bench 基准的具体分数、按量计费的结构性变化、MCP 协议的中立化进程几乎不了解。经过 2026-07-18 的系统检索和信息整理，我对这个赛道的整体格局、关键玩家、成本结构和标准演进有了框架性认知。后续需要通过月度更新和持续实验来把框架细化，同时补上开源 Agent（Cline/OpenCode）的实际使用体验。
