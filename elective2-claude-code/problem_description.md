# 问题描述

## 原始问题

我每个月在 AI 编程工具上花了多少钱？每次 API 调用消耗了多少 Token？在按量计费成为行业标准的 2026 年，不追踪 Token 消耗就等于不知道每个月的话费账单——但这不是钱的问题（对学生来说每月 ¥50 封顶），而是**如果不记录，就永远不会发现自己的使用效率是否在提升**。

## 用自然语言向 Claude Code 描述的任务

> 帮我做一个 AI Token 成本仪表盘页面。功能需求：
> 1. 顶部显示统计卡片：总调用次数、总 Token 消耗、总成本、日均成本
> 2. 一个折线图，显示每天的 Token 消耗趋势
> 3. 一个饼图，显示不同模型的成本分布
> 4. 一个表格，显示所有使用记录
> 5. 一个表单，可以手动录入新的记录
> 6. 数据存储在浏览器 localStorage 里
> 7. 内置我之前的真实实验数据
> 8. 同时写一个 Node.js 命令行脚本，方便在终端直接记录

## 为什么这个问题有意义

按量计费时代，Token = 钱。做一个仪表盘的目的不是"有个漂亮页面"，而是：
- 知道每次调用花了多少钱
- 能对比不同模型的实际性价比
- 月预算 ¥50 是否真的够用 → 用数据而不是感觉

这和我的 AI 信息壁垒报告的结论一致：成本意识本身就是一种竞争优势。
