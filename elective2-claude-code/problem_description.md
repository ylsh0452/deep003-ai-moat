# 真实问题说明

## 1. 我遇到的真实问题是什么

在选修任务一（AI 信息壁垒报告）中，我发现了一个核心结论：按量计费时代，"会用"和"不会用"的人成本差 10 倍以上。但尴尬的是——**我自己都不知道自己每次 API 调用到底花了多少钱**。报告里写了"月预算 ¥50 封顶""DeepSeek API 约 ¥2/百万 Token"，但这些数字是我从二手报道里引用的，还是我自己实际验证过的？没有记录就没有答案。我需要一个工具来追踪和可视化 Token 消耗。

## 2. 这个问题为什么我自己不容易解决

做这个仪表盘需要：写 HTML 结构、写 CSS 样式（还要好看）、用 Chart.js 画折线图和饼图、写 localStorage 读写逻辑、写表单交互——我自己的前端能力不足以独立完成这些。单是 Chart.js 的配置项就够我查半天文档。但我清楚地知道"我想要什么"——一个能看趋势、能记成本、打开就能用的仪表盘。这个"知道要什么但做不出来"的差距，正好是 AI 编程工具应该填补的。

## 3. 为什么适合用 Claude Code 解决

三个原因：
- 这是一个**需求明确、实现标准化**的任务——仪表盘、图表、表单都是成熟的前端模式，AI 生成的质量高
- **我不需要从零学 Chart.js 配置**，我只需要描述"显示一条折线，X 轴是日期，Y 轴是 Token 数"，AI 会自己写出正确的配置
- Claude Code 可以直接**生成完整文件 + 本地验证**——不是聊天窗口里贴一段代码让我自己组装，而是直接写入文件系统，我打开浏览器就能看效果

## 4. 我希望最终得到什么结果

一个可以直接打开的 HTML 文件，打开后看到：
- 统计卡片（调用次数 / Token / 成本 / 日均）
- 一个正常渲染的折线图和一饼图
- 一个可用的录入表单
- 数据刷新后不丢失

再加上一个命令行脚本，输 `node log.js "做了xxx" 300 500` 就能记录一条。最终成果推送到 GitHub，任何人打开仓库都能看到代码、截图和说明。

## 5. 这个任务和我的 AI 壁垒有什么关系

直接延续了选修任务一的行动建议。任务一的分析告诉我两件事：(1) 成本意识是信息壁垒的组成部分，(2) "项目从 Day 1 加入成本监控"是具体的行动项。这个仪表盘就是那个行动项的落地产物——它不是独立作品，它是我 AI 信息雷达持续运转的配套基础设施。以后每次用 Agent 做完任务，用 `log.js` 记一笔，打开仪表盘就能看到趋势——这就是从"知道"到"做到"。
