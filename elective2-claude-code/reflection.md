# 任务复盘

## 1. 我解决了一个什么真实问题

在选修任务一（AI 信息壁垒报告）中，我发现了一个核心结论：按量计费时代，"会用"和"不会用"的人成本差 10 倍以上。但问题来了——**我根本不知道我自己每次调用花了多少钱**。报告里写"月预算 ¥50 封顶"，但这个数字是我估计的还是有数据支撑的？没有记录就没有答案。

这个仪表盘就是解决这个缺口。做完之后，打开页面就能看到：5 次调用、约 11,700 tokens、总成本约 ¥0.02——原来 AI 编程的成本低到这个程度，和我之前"每月花几十美元"的模糊认知完全不同。

## 2. Claude Code 做了哪些我做不到的事

- **一次性生成 280 行可运行代码**：我自己手写需要 2-3 小时，Claude Code 用了约 40 秒。
- **内置 7 个模型的定价表**：它从自己的知识里调出了公开定价数据，我只需要核实，不需要逐一去每个官网查。
- **写了一个我描述时完全没说到的东西**：`log.js` 脚本是我要求的功能之外的——我描述了"仪表盘"，它自己加上了"命令行日志工具"。这说明了自然语言描述的灵活性。

## 3. 本次任务的完整 pipeline

```
模糊需求 → 自然语言描述 → AI 生成代码 → 本地验证 → 
发现报错 → AI 定位修复 → 截图验收 → git 提交 → GitHub 可查
```

这条 pipeline 里的每一步都可以复用。下次做 MCP Server 项目时，同样的流程直接套用。

## 4. 做得好的

- **需求描述具体**：没有说"做个好看的仪表盘"，而是列出了"统计卡片 / 折线图 / 饼图 / 表格 / 表单"五个功能点
- **验收不敷衍**：不是生成完就提交——实际打开了页面、跑了脚本、加了数据、截了图
- **保留了踩坑记录**：git 提交历史完整展示了 submodule 问题的发现和修复过程

## 5. 需要改进的

- **截图数量只满足最低要求**：目前只有 1 张仪表盘截图，如果能加一张命令行 `log.js` 运行截图会更好
- **CSS 和 JS 内联在 HTML 里**：后续功能增多时应该拆分
- **没有做移动端适配**：目前只在桌面端测试过

## 6. 这次任务和普通 AI 用户有什么区别

普通 AI 用户：打开 Claude → 问一个问题 → 复制答案 → 关闭。产出是一个聊天记录。

这次任务：描述需求 → AI 生成项目文件 → 本地验证 → 报错修复 → 截图 → README → 过程记录 → git 推送。产出是一个可运行、可查看、可复现的完整项目，放在 GitHub 上任何人都能看。

差距不在于"AI 厉不厉害"，而在于**是否把 AI 的输出推进到了可交付的终点**。
